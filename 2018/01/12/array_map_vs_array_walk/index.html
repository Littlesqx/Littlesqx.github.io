array_map 和 array_walk 的对比

Posted by Littlesqx on January 12, 2018

好颓好颓~

array_map

array_map — 为数组的每个元素应用回调函数。

array array_map ( callable $callback , array $array1 [, array $… ] )

array_walk

array_walk — 使用用户自定义函数对数组中的每个元素做回调处理

array_walk ( array &$array , callable $callback [, mixed $userdata = NULL ] )

相同点

虽然官方文档解释有点区别（回调函数 / 自定义函数做回调），其实意思都一样。

不同点

参数

  • array_map

    先传回调函数，再传数组参数。

    callback 函数形参（都是数组）的数量和传给 array_map() 数组数量，两者必须一样。

  • array_walk

    先传数组参数（引用），再传回调函数。

    callback 函数默认接受两个参数。array 参数的值作为第一个，键名作为第二个。

    如果提供了可选参数 userdata，将被作为第三个参数传递给 callback 函数。

返回值

  • array_map

    返回数组（callback 函数处理之后）。

  • array_walk

    返回布尔值（成功则 TRUE ，失败则 FALSE）。

示例

1# array_map 一般用法

<?php

$arr1 = [
	'a' => 'test1',
	'b' => 'test2',
	'c' => 'test3'
];

$data = array_map(function ($item) {
	return 'new_' . $item;
}, $arr1);

var_dump($data);

var_dump($arr1);

上面的打印结果，$data　是新数组，而　$arr1　还是旧数组。

2# array_walk 一般用法

<?php

$arr1 = [
	'a' => 'test1',
	'b' => 'test2',
	'c' => 'test3'
];

$arr2 = $arr1;

$flag = array_walk($arr2, function (&$item, $key) {
	$item = 'new_' . $item;
});

if ($flag) {
	var_dump($arr2);
}

$arr2 是新数组。

3# array_map 也传引用呢？

<?php
$arr1 = [
	'a' => 'test1',
	'b' => 'test2',
	'c' => 'test3'
];

$data = array_map(function (&$item) {
	$item = 'new_' . $item;
	return $item;
}, $arr1);

var_dump($arr1);
var_dump($data);

结果和 1# 一样。
