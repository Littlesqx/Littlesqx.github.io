设计模式 (1) - 单例模式

Posted by Littlesqx on January 7, 2018

好颓好颓~

简单介绍

单例模式保证了应用在生命周期间一个类只生成一个实例对象，实现思路是类只能返回对象一个引用和一个获得该实例的方法。它的好处是避免频繁重复实例化同一个类，而造成系统资源的消耗和浪费。

PHP 实现

一个栗子

直接上代码。

<?php

class Product {

	private static $instance;

	public static function getInstance(): Product
	{
		if (! self::$instance) {
			self::$instance = new self();
		}
		return self::$instance;
	}
}

$product = Product::getInstance();
$product2 = Product::getInstance();

var_dump($product === $product2);

上面程序的运行结果是：

bool(true)

说明了 $product 实例和 $product2 实例是同一个引用。

改进一下

上面的例子虽然说是已经实现了单例模式的效果，但还是可以直接通过 new Product() 的方式实例化对象，又或者使用 clone 关键字复制对象，这样都会重新生成新对象，违反了单例模式中实例唯一的原则。

<?php

class Product {

	private static $instance;

	public static function getInstance(): Product
	{
		if (! self::$instance) {
			self::$instance = new self();
		}
		return self::$instance;
	}
    
        // 防止直接 new Product() 生成对象

	private function __construct() {}
    
        // 防止直接 clone $product 生成对象

	private function __clone() {}

}

栗子升级

如果每个类都需要实现单例模式，那不是每个类都重写一遍这些代码？当然不能这样做，也当然有办法解决这个问题。

方法一、使用 Trait

Trait 是为类似 PHP 的单继承语言而准备的一种代码复用机制。

<?php

trait Singleton {

	private static $instance;

	public static function getInstance()
	{
		if (! self::$instance) {
			self::$instance = new self();
		}
		return self::$instance;
	}

	private function __construct() {}

	private function __clone() {}

}

class Pan {

	use Singleton;

	private $name = 'Pan';

	public function getName(): string
	{
		return $this->name;
	}
}

class Book {

	use Singleton;

	private $name = 'Book';

	public function getName(): string
	{
		return $this->name;
	}
}

echo Pan::getInstance()->getName(), "\n";

echo Book::getInstance()->getName(), "\n";

上面的程序的运行结果是：

Pan
Book

方法二、使用继承

这里面涉及到了后期静态绑定， static 关键字的使用。同时，存实例的 $instance 变量变成了数组类型，用于管理多个子类的单例。

<?php

abstract class Product {

	private static $instances;

	private $name;

	public static function getInstance()
	{
                // 也可以使用 $class = static::class;
		$class = get_called_class();
		if (! isset(self::$instances[$class])) {
                        // 也可以使用 new $class;
			self::$instances[$class] = new static();
		}
		return self::$instances[$class];
	}

	public abstract function getName();
}


class Pan extends Product {

	private $name = 'Pan';

	public function getName(): string
	{
		return $this->name;
	}
}

class Book extends Product {

	private $name = 'Book';

	public function getName(): string
	{
		return $this->name;
	}
}

echo Pan::getInstance()->getName(), "\n";

echo Book::getInstance()->getName(), "\n";

同样地，上面的程序输出也是：

Pan
Book

补充

PHP 语言内大概就这样，但在其他多线程语言（比如 JAVA ）就需要更加小心。当两个线程并发请求生成，就有可能重复生成实例，所以需要给生成实例的代码加互斥锁，使得其运行能够线程同步。

使用 synchronized 关键字便是最常用的方法：

public class Singleton {  
    private static Singleton instance;  
    private Singleton (){}  
  
    public static synchronized Singleton getInstance() {  
        if (instance == null) {  
            instance = new Singleton();  
        }  
        return instance;  
    }  
}
