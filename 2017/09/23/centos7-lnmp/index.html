在CentOS 7 上配置 LNMP 环境

Posted by Littlesqx on September 23, 2017

终于开始了第一篇博客，希望自己可以坚持下去。内容是基本的环境配置，但尽量补充必要知识。

前言

提到php的运行环境，就想起了第一次接触php，是社团师兄带我们入门web开发。师兄推荐的xampp，类似的还有WampServer，都是windows下集成好的php开发环境（包括Apache，MySQL，还有PHP）。很方便，一键式的配置，之后就可以把时间集中在php代码实现上。但是缺点也很明显，直接跳过了web环境的原理，失去了学习一些必要的知识。就我自己来说，很久之后才对服务器有大体的正确认识。可能还有其他的集成环境，但是都离不开LNMP和LAMP这两种架构。查阅相关资料的过程中，发现还有LNAMP，有空再写一篇他们之间的介绍和对比文章。

正文

配置源

1# 查看系统版本

[root@VM_157_248_centos ~]# cat /etc/centos-release
CentOS Linux release 7.4.1708 (Core)
[root@VM_157_248_centos ~]#

2# 配置MariaDB官方源

根据MariaDB官方源，选择系统合适的版本 CentOS > CentOS 7(x86_64) > 10.2 [Stable] ：

# MariaDB 10.2 CentOS repository list - created 2017-09-23 09:53 UTC
# http://downloads.mariadb.org/mariadb/repositories/
[mariadb]
name = MariaDB
baseurl = http://yum.mariadb.org/10.2/centos7-amd64
gpgkey=https://yum.mariadb.org/RPM-GPG-KEY-MariaDB
gpgcheck=1

vi /etc/yum.repos.d/MariaDB.repo
# 填入前面的代码段，ESC :wq 保存退出

3# 配置Nginx官方源 根据Nginx官方源，选择系统合适的版本

[nginx]
name=nginx repo
baseurl=http://nginx.org/packages/centos/7/$basearch/
gpgcheck=0
enabled=1

vi /etc/yum.repos.d/nginx.repo
# 填入前面的代码段，ESC :wq 保存退出

4# 配置PHP源

rpm -Uvh https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
rpm -Uvh https://mirror.webtatic.com/yum/el7/webtatic-release.rpm

安装

1# 安装MariaDB

yum install MariaDB-server -y

2# 安装Nginx

yum install nginx -y

3# 安装PHP和常用扩展

yum install php71w-cli php71w-fpm -y
yum install php71w-mbstring php71w-pdo -y
yum install php71w-gd php71w-mcrypt -y
yum install php71w-mysql php71w-xml -y
yum install php71w-soap php71w-xmlrpc -y  
yum install php71w-opcache -y
# 其他
yum search php71w

配置

1# 设置MariaDB

mysql_secure_installation
# 默认密码为空，要求键入旧密码时回车即可
# 拒绝远程访问。如果实在需要外网连接，设置允许单独的ip
GRANT ALL PRIVILEGES ON [数据库.]* TO '用户名'@'允许访问的IP地址' IDENTIFIED BY '密码' WITH GRANT OPTION;
FLUSH PRIVILEGES；

2# 设置PHP

vi /etc/php.ini

找到 ;cgi.fix_pathinfo=1 （可通过 /cgi.fix_pathinfo 搜索）

修改成 cgi.fix_pathinfo=0

原因可参考Nginx + PHP CGI的一个可能的安全漏洞

3# 配置Nginx

vi /etc/nginx/conf.d/default.conf

修改以下代码段，配置正确的路径

# pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000
#
location ~ \.php$ {
    root           /usr/share/nginx/html;
    fastcgi_pass   127.0.0.1:9000;
    fastcgi_index  index.php;
    fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
    include        fastcgi_params;
}

测试

systemctl restart nginx
systemctl restart php-fpm

结尾

关于性能

PHP7刚出来时就以性能为主打，确实惊艳了不少PHPer。除了php7本身，鸟哥在自己博客分享了其他的几个技巧

1# 启用Zend Opcache （貌似7.1是默认开启）

vi /etc/php.ini

加入以下代码段

zend_extension=opcache.so
opcache.enable=1
opcache.enable_cli=1"

ESC :wq 保存退出

2# 使用新的编译器 gcc version >= 4.8

3# HugePage

sysctl vm.nr_hugepages=512

vi /etc/php.ini

加入以下代码段

opcache.huge_code_pages=1

ESC :wq 保存退出
