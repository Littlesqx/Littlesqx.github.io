Go defer 关键字

Posted by Littlesqx on January 12, 2018

好颓好颓~

defer 关键字

Go 语言中使用 defer 关键字在函数中声明延迟( defer )语句，顾名思义，这些语句会被逆序延迟执行。 执行时间是函数返回(return)前。

栗子：

func main() {
    defer fmt.Printf("a ")
    defer fmt.Printf("b ")
    fmt.Printf("c ")
}

输出 c b a

栗子：

func ReadWrite() bool {
    file.Open("file")
    // do something...
    if failureX {
        file.Close()
        return false
    }

    if failureY {
        file.Close()
        return false
    }

    file.Close()
    return true
}

在一些打开资源的函数，如果遇到错误应及时关闭相关资源，以避免某些资源泄漏问题。但上面的代码关闭语句重复较多，使用 defer 语句可以更加优雅。

func ReadWrite() bool {
    file.Open("file")
    defer file.Close()
    // do something...
    if failureX {
        return false
    } else if failureY {
        return false
    }
    return true
}

defer 使用时的坑

栗子：

func f1() (result int) {
    defer func() {
        result++
    }()
    return 0
}
func f2() (result int) {
     t := 5
     defer func() {
       t = t + 5
     }()
     return t
}
func f3() (r int) {
    defer func(r int) {
          r = r + 5
    }(r)
    return 1
}

f1()，f2()，f3() 的输出结果分别是1，５，１。

关键在于 return xxx 这一条语句并不是一条原子指令!

函数返回的过程是这样的：先给返回值赋值，然后调用defer表达式，最后才是返回到调用函数中。

那上面的栗子，就可以转换为：

func f1() (result int) {
    result = 0 // 0 来源于 `return 0` ，非未初始化的默认值
    defer func() {
        result++
    }()
    return 0
}
func f2() (result int) {
     t := 5 // // 0 来源于 `return t`
     result = t
     defer func() {
       t = t + 5
     }()
     return t
}
func f3() (r int) {
    r = 1 // // 0 来源于 `return 1`
    defer func(r int) {
          r = r + 5
    }(r)
    return 1
}

f3 的例子闭包传的是形参，所以 r 的改变不影响返回值。
