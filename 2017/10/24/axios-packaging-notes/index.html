前言

正文

axios

web开发里，axios是首推的Ajax库了，应该说基本上都用它。

官方是这样介绍的：

Make XMLHttpRequests from the browser

Make http requests from node.js

Supports the Promise API

Intercept request and response

Transform request and response data

Cancel requests

Automatic transforms for JSON data

Client side support for protecting against XSRF

谈谈流程

一个体验好的web应用，应该具备的请求过程有什么？

pic# 对，我又画了个图

确定参数：请求url，请求方式，请求参数，其他请求参数（时限？请求头？）

过渡动画：请求开始后一直到请求结束之间有过渡动画，容易想到axios提供的拦截器

视图响应：提示框？对话框？路由跳转？

怎么实现

确定参数

1# 接口是多样的，接口的代码入口，是以url还是method来区分？

其实都有，以url为区分，会设计为每个接口一个函数；以method为区分，一般只有涉及到的get，post，put，patch，delete，那就主要有这么几个函数，实际调用再具体指定url。

个人倾向于后者，应为当借口类型多了，那前者写比较多函数，有点多余了。但是后者有个问题，当url变动时，每一处调用都需要修改。参考工厂模式的思想，可以把url写到单个配置文件，这样就可以避免了。

2# 动态的参数怎么设置？比如验证token，接受语言。

这部分配置都在请求头处，那可以增加header参数，因为axios已提供动态修改或header的api。

一些公共的header，比如接受语言，那就可以利用拦截器增加。

过渡动画

axios的拦截器包括了请求前的拦截和响应后的拦截。那过渡动画的增加就简单了。

视图响应

还是选择拦截器，可根据响应结果做具体处理。

代码样例

我的是用在vue上， ./utils/http.js

import qs from 'qs'
import axios from 'axios'
import store from '../store'

axios.defaults.headers.common['X-Requested-With'] = 'XMLHttpRequest'
axios.defaults.timeout = 10000 // ms
axios.defaults.baseURL = '/api'
axios.defaults.headers = {
    "Content-Type": "application/x-www-form-urlencoded",
    'accept-language': store.getters.lang
}
// cors with cookie
// axios.defaults.withCredentials = true

axios.interceptors.request.use(config => {

    // set configs
    config.headers['accept-language'] = store.getters.lang

    // loading start
    // ...

    return config
}, error => {
    return Promise.reject(error)
})

axios.interceptors.response.use(response => {
    return response
}, error => {

    const { response } = error

    // loading finish
    // ...

    // deal with errorResponse
    // ...

    return Promise.reject(error)
})

export default {
    get(url, params, header = {}) {
        return axios.get(url, {
            params, header
        })
    },
    post(url, params, header = {}) {
        return axios.post(url, qs.stringify(params), {
            header
        })
    },
    put(url, params, header = {}) {
        return axios.put(url, qs.stringify(params), {
            header
        })
    },
    patch(url, params, header = {}) {
        return axios.patch(url, qs.stringify(params), {
            header
        })
    },
    delete(url, params, header = {}) {
        return axios.delete(url, {
            header, params
        })
    }
}

调用代码

import http from './utils/http'

fetch() {
  http.get(url, params).then( response => {
    // ...
  }).catch( error => {
    // ...
  })
}

有必要使用async/await代替Promise

import http from './utils/http'

async fetch() {
  try {
    let response = await http.get(url, params)
    // ...
  } catch (error) {
    // ...
  }
}

使用async/await有效避免很多回调的代码，线性的执行更好理解。
